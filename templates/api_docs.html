Integra tu sistema con nuestra plataforma de recargas

Autenticación

Todas las peticiones requieren tu API Key en el header:

X-API-Key: TU_API_KEY_AQUI

Obtén tu API Key en Mi API. También puedes enviarla como parámetro ?api_key=TU_KEY

URL Base

{{ request.host_url }}api/v1/

Tipos de Producto

Hay dos tipos de productos que se comportan diferente al comprar:

TipoRequiereResultadoEjemplo
Recarga directa id_juego (Player ID) Diamantes/créditos aplicados a la cuenta Mobile Legends, PUBG, Free Fire
Gift Card / Código Nada Se entrega un código Roblox, Steam, PlayStation, iTunes

Usa /api/v1/productos para ver qué campos requiere cada producto en campos_requeridos.

POST

/api/v1/comprar

Realiza una compra. La recarga o entrega del código se ejecuta automáticamente.

Parámetros (JSON Body):

CampoTipoRequeridoDescripción
producto_idintegerSíID del producto a comprar
cantidadintegerNoCantidad (default: 1)
id_juegostringRecargasPlayer ID / User ID del jugador. Corresponde al primer campo de campos_requeridos
input2stringSegún juegoZona / Servidor. Corresponde al segundo campo de campos_requeridos (si existe)

Ejemplo: Recarga directa (Mobile Legends)

curl -X POST \
  -H "X-API-Key: TU_KEY" \
  -H "Content-Type: application/json" \
  -d '{"producto_id": 1, "id_juego": "123456789", "input2": "10521"}' \
  {{ request.host_url }}api/v1/comprar

Respuesta exitosa (recarga):

{
  "ok": true,
  "pedido_id": 42,
  "estado": "completado",
  "total": 1.50,
  "saldo_restante": 148.50,
  "nombre_jugador": "ProGamer123",
  "referencia": "GP-20260318-001",
  "mensaje": "Recarga completada para ProGamer123 (Ref: GP-20260318-001)"
}

Ejemplo: Gift Card (Roblox)

curl -X POST \

Respuesta: Pedido en procesamiento (recarga manual)

Algunos productos requieren procesamiento manual por el proveedor. En estos casos el pedido queda en estado procesando hasta que el proveedor confirme.

{
  "ok": true,
  "pedido_id": 46,
  "estado": "procesando",
  "total": 2.50,
  "saldo_restante": 147.50,
  "referencia": "GP-20260319-003",
  "mensaje": "Pedido enviado al proveedor (Ref: GP-20260319-003). Se confirmará automáticamente."
}

Pedido en procesamiento: El estado cambiará automáticamente a completado o cancelado cuando el proveedor responda (generalmente en minutos). Si se cancela, el saldo se reembolsa automáticamente. Puedes consultar el estado con /api/v1/pedido/:id o recibir notificación vía Webhook.

Respuesta error (con reembolso automático):

{
  "ok": false,
  "error": "Player ID inválido",
  "pedido_id": 43,
  "reembolsado": true,
  "saldo_restante": 150.00
}

Reembolso automático: Si la recarga falla, el saldo se devuelve automáticamente a tu cartera. El campo reembolsado: true confirma el reembolso.

Webhooks (Notificaciones automáticas)

Registra una URL de webhook para recibir notificaciones automáticas cuando un pedido cambie de estado (de procesando a completado o cancelado). Así no necesitas consultar periódicamente.

Registrar / Actualizar webhook:

POST /api/v1/webhook 
curl -X POST \
  -H "X-API-Key: TU_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://tu-pagina.com/callback"}' \
  {{ request.host_url }}api/v1/webhook
{
  "ok": true,
  "mensaje": "Webhook registrado: https://tu-pagina.com/callback"
}

Consultar webhook actual:

GET /api/v1/webhook 
{
  "ok": true,
  "webhook_url": "https://tu-pagina.com/callback"
}

Eliminar webhook:

POST /api/v1/webhook
Body: {"url": ""}

Notificación: Pedido completado

Tu sistema recibirá un POST con este JSON cuando el proveedor confirme la recarga:

{
  "evento": "pedido_actualizado",
  "pedido_id": 46,
  "estado": "completado",
  "referencia": "GP-20260319-003",
  "nombre_jugador": "ProGamer123"
}

Notificación: Pedido cancelado (con reembolso)

Si el proveedor rechaza la recarga, recibirás esta notificación y el saldo se reembolsa automáticamente:

{
  "evento": "pedido_actualizado",
  "pedido_id": 46,
  "estado": "cancelado",
  "referencia": "GP-20260319-003",
  "razon": "Invalid player ID",
  "reembolso": 2.50
}

Nota: El webhook es opcional. Si no registras una URL, puedes consultar manualmente el estado con GET /api/v1/pedido/:id. Tu endpoint debe responder con HTTP 200 para confirmar la recepción.

Estados de Pedido

EstadoDescripciónAcción
completado Recarga aplicada o código entregado exitosamente Entregar al cliente
procesando Enviado al proveedor, esperando confirmación Esperar webhook o consultar /api/v1/pedido/:id
cancelado Rechazado por el proveedor. Saldo reembolsado automáticamente Informar al cliente que la recarga no fue posible

Códigos HTTP

CódigoSignificado
200Petición exitosa
400Error en la petición (saldo insuficiente, error de recarga, campo faltante)
401API Key faltante o inválida
404Producto o pedido no encontrado
429Demasiadas peticiones (rate limit)
500Error interno del servidor

Flujo Recomendado

  1. Registra tu webhook con POST /api/v1/webhook (una sola vez)
  2. Consulta /api/v1/productos para ver el catálogo y los campos requeridos por producto
  3. Consulta /api/v1/saldo para verificar que tienes saldo suficiente
  4. Envía POST /api/v1/comprar con el producto_id y campos correspondientes
  5. Si estado es "completado": la recarga ya fue aplicada o el código fue entregado
  6. Si estado es "procesando": espera la notificación en tu webhook
  7. Cuando recibas el webhook con "estado": "completado", entrega al cliente
  8. Si recibes "estado": "cancelado", el saldo ya fue reembolsado. Informa al cliente
  9. También puedes consultar el estado manualmente con /api/v1/pedido/:id

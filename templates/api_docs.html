Documentación API v1

Integra tu sistema con nuestra plataforma de recargas y gift cards

Inicio Rápido — Tu primera compra en 3 pasos

Sigue estos 3 pasos para hacer tu primera recarga o compra de gift card por API:

1
Obtén tu API Key

Ve a Mi API y copia tu clave. Esa clave va en todas las peticiones.

2
Consulta los productos

Llama a GET /api/v1/productos para ver qué se vende, los precios y qué datos necesita cada producto.

3
Compra

Llama a POST /api/v1/comprar con el producto_id y listo. Si es recarga, envía también el id_juego del cliente. Si es gift card, recibes los códigos en la respuesta.

Ejemplo rápido — Comprar gift card en PHP:

// Comprar una gift card de Roblox
$ch = curl_init('{{ request.host_url }}api/v1/comprar');
curl_setopt_array($ch, [
    CURLOPT_POST => true,
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => [
        'X-API-Key: TU_API_KEY',
        'Content-Type: application/json'
    ],
    CURLOPT_POSTFIELDS => json_encode([
        'producto_id' => 5,
        'cantidad' => 1
    ])
]);
$resp = json_decode(curl_exec($ch), true);
curl_close($ch);

if ($resp['ok']) {
    // Entrega estos códigos a tu cliente
    foreach ($resp['codigos'] as $codigo) {
        echo "Código: $codigo\n";
    }
} else {
    echo "Error: " . $resp['error'];
}

Autenticación

Todas las peticiones requieren tu API Key. Puedes enviarla de dos formas:

Opción 1 — Header (recomendado):
X-API-Key: TU_API_KEY_AQUI
Opción 2 — Query param:
{{ request.host_url }}api/v1/saldo?api_key=TU_API_KEY_AQUI

Obtén tu API Key en Mi API.

URL Base

{{ request.host_url }}api/v1/

Todas las respuestas son JSON. Envía Content-Type: application/json en las peticiones POST.

Tipos de Producto

Hay 3 tipos de productos. Cada uno funciona diferente al comprar:

TipoQué enviarQué recibesJuegos
Recarga directa id_juego + input2 (si aplica) Los diamantes/créditos se aplican directo a la cuenta del jugador Mobile Legends, PUBG, Blood Strike, Genshin
Recarga con PIN id_juego del jugador Se aplica la recarga usando PINes internos (puede ser multi-canje) Free Fire
Gift Card / Código Solo producto_id y opcionalmente cantidad Recibes un array codigos con los códigos para entregar a tus clientes Roblox, Steam, PlayStation, iTunes

Tip: ¿Cómo saber qué tipo es? Consulta /api/v1/productos. Si campos_requeridos está vacío = gift card. Si tiene campos = recarga (debes enviar id_juego).

POST

/api/v1/comprar

Realiza una compra. La recarga o entrega del código es automática. El costo se descuenta de tu saldo.

Parámetros (JSON Body):

CampoTipoRequeridoDescripción
producto_idintegerSiempreID del producto (obtenido de /productos)
cantidadintegerNoCantidad de códigos (solo gift cards, default 1, máx 50). Para recargas siempre es 1.
id_juegostringSolo recargasEl Player ID / User ID del cliente (primer campo de campos_requeridos)
input2stringSegún juegoZona o servidor (segundo campo de campos_requeridos, si existe)

Algunos productos de Free Fire usan multi-canje (consumen varios PINes internos). Tú solo envías el ID, el sistema hace todo:

Body: {"producto_id": 8, "id_juego": "987654321"}

Respuesta exitosa:

{
  "ok": true,
  "pedido_id": 60,
  "estado": "completado",
  "total": 3.00,
  "saldo_restante": 147.00,
  "nombre_jugador": "FirePlayer99",
  "canjes_realizados": 3,
  "mensaje": "3 recarga(s) aplicada(s) a FirePlayer99 (ID: 987654321)"

Canje parcial: Si un producto necesita 3 canjes y solo 2 funcionan, recibes "canjes_realizados": 2, "canjes_esperados": 3, "reembolso_parcial": 1.00. La parte no completada se reembolsa automáticamente.

Respuesta: Pedido en procesamiento

Algunos productos (como Honor of Kings, PUBG UC y otros marcados con procesamiento_manual: true) requieren confirmación del proveedor. La recarga se realiza pero el proveedor tarda unos minutos en confirmar.

¿Cómo funciona?

  1. Tu petición POST /api/v1/comprar envía la recarga al proveedor.
  2. El sistema espera hasta 30 segundos para que el proveedor confirme.
  3. Si el proveedor confirma en esos 30 segundos → recibes "estado": "completado" directamente.
  4. Si aún no confirma → recibes "estado": "procesando". La recarga ya se hizo, solo falta la confirmación.
  5. Nuestro sistema verifica automáticamente cada minuto y actualiza el estado a "completado" en ~5 minutos.
  6. Cuando cambie de estado, recibes una notificación en tu webhook (si lo tienes configurado).
{
  "ok": true,
  "pedido_id": 46,
  "estado": "procesando",
  "total": 2.50,
  "saldo_restante": 147.50,
  "referencia": "GP-20260319-003",
  "mensaje": "Pedido enviado al proveedor (Ref: GP-20260319-003). Se confirmará automáticamente."
}

¿Qué hacer? Espera el webhook o consulta GET /api/v1/pedido/46 después de 5 minutos. El estado cambiará a "completado" automáticamente. Si falla (raro), cambia a "cancelado" y tu saldo se reembolsa.

Tip: Para no tener que consultar manualmente, registra un webhook con POST /api/v1/webhook. Recibirás una notificación automática cuando el pedido pase a "completado" o "cancelado".

Webhooks (Notificaciones automáticas)

Registra una URL para recibir notificaciones automáticas cuando un pedido "procesando" cambie de estado. Así no necesitas consultar manualmente.

Registrar webhook:

POST /api/v1/webhook 
Body: {"url": "https://tu-sistema.com/callback"}
{
  "ok": true,
  "mensaje": "Webhook registrado: https://tu-sistema.com/callback"
}

Ver / Eliminar webhook:

GET /api/v1/webhook para ver tu URL actual. Para eliminar, envía POST con {"url": ""}.

Tu sistema recibe esto cuando se completa:

{
  "evento": "pedido_actualizado",
  "pedido_id": 46,
  "estado": "completado",
  "referencia": "GP-20260319-003",
  "nombre_jugador": "ProGamer123"
}

Tu sistema recibe esto si se cancela:

{
  "evento": "pedido_actualizado",
  "pedido_id": 46,
  "estado": "cancelado",
  "referencia": "GP-20260319-003",
  "razon": "Invalid player ID",
  "reembolso": 2.50
}

Nota: El webhook es opcional. Si no lo configuras, consulta el estado con GET /api/v1/pedido/:id. Tu endpoint debe responder HTTP 200 para confirmar recepción.

Flujo Recomendado para tu Sistema

  1. Configuración inicial (una vez): Obtén tu API Key y registra tu webhook si quieres notificaciones automáticas.
  2. Sincroniza catálogo: Consulta /api/v1/productos para mostrar los productos a tus clientes.
  3. Antes de comprar: Verifica saldo con /api/v1/saldo.
  4. Compra: Tu cliente elige producto y te da su Player ID (si aplica). Envías POST /api/v1/comprar.
  5. Si estado = "completado": Entrega al cliente los códigos o confirma la recarga.
  6. Si estado = "procesando": Dile al cliente que espere. El webhook te avisará cuando esté listo.
  7. Si hay error con reembolsado = true: Tu saldo ya fue devuelto. Informa al cliente.

Preguntas Frecuentes

¿Cómo recargo mi saldo?

Contacta al administrador. El saldo se acredita manualmente a tu cuenta.

¿Qué pasa si mando un Player ID incorrecto?

Si el proveedor rechaza la recarga, tu saldo se reembolsa automáticamente. Verás "reembolsado": true en la respuesta.

¿Puedo comprar varias gift cards a la vez?

Sí. Envía "cantidad": 5 para recibir 5 códigos en una sola petición (máximo 50). Cada código es independiente.

¿Qué es "procesando"?

Significa que el proveedor está procesando tu pedido. Generalmente toma segundos a minutos. Configura un webhook para recibir la notificación automáticamente.

¿Necesito webhook obligatoriamente?

No. Es opcional pero recomendado. Sin webhook puedes consultar el estado de cada pedido con GET /api/v1/pedido/:id.

¿Cuál es el límite de peticiones?

No hay un límite estricto, pero evita hacer más de 60 peticiones por minuto. Si recibes error 429, espera unos segundos y reintenta.
